# Composition Patterns and Originality Rules

## Basic Structure Types

Choose one structure only. Do not mix too many.

### Workflow

Best for input -> process -> output, content production, AI workflows, and automation chains.

Layout: input on the left, Xiaohei or a strange machine processing in the middle, output on the right, with orange arrows showing the main flow.

### System Fragment

Best for information sources, filters, databases, renderers, and agent subsystems.

Layout: draw only 3-5 core modules, with Xiaohei taking one key action.

### Before/After Contrast

Best for messy/ordered, manual/automatic, scattered/collected, anxious/stable.

Layout: messy left, stable right, orange arrow in the middle. Characters can be more exaggerated.

### Character State

Best for pain points, creator state, too many tools, information anxiety, and stuck -> moving.

Layout: 2-4 small states, each with a short label.

### Conceptual Metaphor

Best for content factories, information warehouses, brain black boxes, workflow machines, and auto reports.

Layout: one big strange object or machine, a few inputs, one output. It needs a memorable image.

### Layered Method

Best for frameworks, system layers, capability stacks, and content system layers.

Layout: stacked boxes, not a formal pyramid; Xiaohei can build nearby or move bricks.

### Route Map

Best for idea to launch, user path, content handoff path, and learning routes.

Layout: one curving path, a few nodes, Xiaohei walking or pulling a thread.

### Small Comic Sequence

Best for failure to success, real process, jokes, and before/after use cases.

Layout: 2-4 small scenes, each showing one action.

## Original Metaphor Method

Every time, reinvent the metaphor from the current article. Do not copy old images.
